# 플랜탭 난이도 재배치 + 채널 영상 깊이 분석 설계

**작성일**: 2026-04-28
**상태**: 확정
**근거**: 돈버는하마 채널 26편 영상 분석 + 전략 v2 (`2026-04-27-seller-strategy-v2-design.md`)

---

## 1. 목표

`/plan` 탭의 84개 task를 **난이도별로 재구성**하고 **채널 영상 메시지를 깊이 반영**하여, 매출 1,000만원 달성까지의 학습 곡선을 단계적으로 따라갈 수 있게 한다.

### 결정 요약
- 난이도 시스템: ⭐ 1~5 (5단계)
- 12주 평균 난이도 곡선: 2 → 4 → 3 → 5 (점진+급변+반복+정점)
- 매주 안 정렬: 쉬운 task → 어려운 task (빠른 win 효과)
- 채널 영상 인용: 26편 중 task별 출처 표시

---

## 2. 데이터 구조 확장

`src/lib/plan/constants.ts`의 `WbsTask` 타입 확장:

```ts
export interface WbsTask {
  id: string;
  text: string;
  difficulty: 1 | 2 | 3 | 4 | 5;
  steps?: string[];
  videoRef?: string;
  tip?: string;
  estimatedHours?: number;
}
```

기존 `text`/`steps`/`tip`은 유지. 신규 필드 3개 추가.

## 3. 난이도 기준 (5단계)

| 난이도 | 의미 | 시간/규모 | 대표 예시 |
|---|---|---|---|
| ⭐ | 매우 쉬움 | 30분 이내 | 광고 캠페인 활성화, 메시지 자동응답 등록 |
| ⭐⭐ | 쉬움 | 1~2시간 | 사업자등록, 도매꾹 import 80개, AI 상세페이지 일괄 |
| ⭐⭐⭐ | 보통 | 반나절 | 위너 후보 데이터 분석, 키워드 30개 발굴 |

| ⭐⭐⭐⭐⭐ | 매우 어려움 | 며칠 | 위너 200~300개 스케일 사입, 사업 전략 회고 |

## 4. 12주 평균 난이도 곡선

```
Week 1   ⭐⭐      셋업 (학습 중심)
Week 2   ⭐⭐      등록·광고 시작
Week 3   ⭐⭐⭐    데이터 분석
Week 4   ⭐⭐⭐⭐  ★1688 첫 발주 (난이도 점프)
Week 5   ⭐⭐⭐⭐  사입 검수
Week 6   ⭐⭐⭐    그로스 활성화 (반복 흐름)
Week 7   ⭐⭐⭐⭐  광고 자체조달 + 2차 사입
Week 8   ⭐⭐⭐    CS 자동화 (한 번 셋업)
Week 9   ⭐⭐⭐    인접 영역 확장
Week 10  ⭐⭐⭐⭐⭐ ★대량 사입 (난이도 최고)
Week 11  ⭐⭐⭐⭐  기획전 + SEO
Week 12  ⭐⭐⭐⭐  회고 + 다음 분기 spec
```

### 곡선 설계 원칙
- **Week 1~3**: 부드럽게 진입 (학습 + 첫 매출 50~150만원)
- **Week 4~5**: 첫 점프 (1688 직접 사입 — 자본·외부 협상)
- **Week 6**: 한 번 떨어짐 (그로스 자동화 흐름)
- **Week 7~9**: 안정 (반복 작업)
- **Week 10**: 최고점 (대량 사입 + 광고 최대 + 단일 SKU 일매출 30만원)
- **Week 11~12**: 마무리 (전략적 의사결정)

## 5. 매주 안 task 정렬 원칙

각 주의 task 배열을 **난이도 오름차순**. 사용자가 매주 시작 시 "쉬운 것부터" 처리 → 빠른 win 효과.

예시 (Week 1):
```
1. ⭐  CS 자동 응답 메시지 사전 작성        (등록만)
2. ⭐⭐ 광고 계정 세팅                       (가입+충전)
3. ⭐⭐ 사업자등록증 / 네이버스토어 / 쿠팡윙 가입
4. ⭐⭐ 쿠팡 로켓그로스 진입 신청
5. ⭐⭐⭐ 도매꾹 시드 키워드 30개 발굴 (회피 필터 적용)
6. ⭐⭐⭐ 도매꾹 SKU 80~100개 선별
7. ⭐⭐⭐ SmartSellerStudio /sourcing 워크플로우 숙지
```

## 7. UI 변경 (PlanClient.tsx)

### 7.1 task 표시 구조

```
┌────────────────────────────────────────────────────────┐
│ ☐  도매꾹 시드 키워드 30개 발굴             ⭐⭐⭐  ▶  │
└────────────────────────────────────────────────────────┘
```

- 좌측: 체크박스 (현재처럼 클릭 → 토글)
- 중앙: task 텍스트
- 우측: 별 ⭐⭐⭐ + 확장 버튼 ▶

### 7.2 확장 패널 (▶ 클릭 시)

```
┌────────────────────────────────────────────────────────┐
│ ☑  도매꾹 시드 키워드 30개 발굴             ⭐⭐⭐  ▼  │
│                                                          │
│   📋 단계별 가이드                                       │
│   1. /sourcing 페이지 접속 → 키워드/도매꾹 탭          │
│   2. 시드 키워드 입력 ("주방용품" 등)                  │
│   3. 월 검색 3,000~30,000 / 경쟁상품 500개 미만 체크   │
│   4. 회피 리스트 자동 필터로 차단됨 — 빨간 표시 확인   │
│   5. 엑셀에 키워드 30개 + 검색량 + 경쟁수 기록         │
│                                                          │
│   💡 팁: 검색량보다 "경쟁상품수 ÷ 검색량 < 5%" 기준이  │
│   더 중요. 회피 리스트 통과 못 하는 키워드는 즉시 버림 │
│                                                          │
│   📺 채널 영상: 키워드 찾는법 (2026-03-10)             │
│   ⏱️ 예상 소요: 3시간                                  │
└────────────────────────────────────────────────────────┘
```

### 7.3 색상 코딩
- 별 색상: ⭐ 초록 → ⭐⭐⭐ 노랑 → ⭐⭐⭐⭐⭐ 빨강
- 체크 완료 시 task 전체 회색 + 줄긋기 (현재 동작 유지)

### 7.4 상태 관리
- 확장 상태: `Set<string>` (펼친 task id 집합) — useState
- 체크 상태: 현재 localStorage `plan_wbs_tasks` 그대로 유지

## 9. 작업 순서

1. **WbsTask 타입 확장** + 호환성 (기존 데이터에 difficulty 없으면 default 3)
2. **constants.ts 84 task 일괄 업데이트** — 한 task씩 채널 영상과 매핑하며 difficulty/videoRef 부여
3. **constants.ts 매주 task 정렬** — difficulty 오름차순
4. **PlanClient.tsx UI 변경** — 별 + 확장 버튼 + 확장 패널
5. **빌드 + 동작 확인** — `/plan` 페이지에서 확장 토글, 별 색상, 영상 출처 표시
6. **commit + push** — 자동 재배포

## 10. 다음 단계 (별도 spec)

본 spec은 **플랜탭 task 재구성**만 다룬다. 채널 영상 분석에서 도출된 **11개 신규 기능 후보**(아이템위너 모니터링, 광고 ROAS 자동 알림, 재고 회전 알림 등)는 별도 spec(`2026-MM-DD-strategy-v2-extension-design.md`)에서 다룬다. 본 plan 완료 후 진행.

### 11개 후보 요약 (별도 spec 대상)
- **운영 자동화**: 광고 ROAS 알림 / 재고 회전 알림 / 부정 리뷰 알림
- **위너 관리**: 아이템위너 모니터링 / 위너 SKU 키워드 최적화
- **사입 의사결정**: 위탁 vs 사입 자동 추천 (배치) / 1688 가격 협상 가이드
- **회고/학습**: 회송 사례 추적 DB / CS 자동응답 패턴 분석 / 채널별 분배 모니터링
- **리뷰 관리**: 포토리뷰 적립금 자동화

## 11. 검증 항목

- 84개 task 모두 difficulty 1~5 부여
- 12주 평균 난이도 곡선이 §4 명시 곡선과 일치
- 각 주차 task가 difficulty 오름차순 정렬
- 채널 영상 인용 16개 매핑 task에 videoRef 포함
- /plan 페이지에서 확장 패널 정상 동작
- 색상 코딩이 사용자에게 명확히 전달
- 빌드 성공 + 운영 배포 후 페이지 정상 렌더링
