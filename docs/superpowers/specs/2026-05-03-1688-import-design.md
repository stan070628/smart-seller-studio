## 개요

셀러가 1688에서 저장한 상품 이미지를 업로드하면 Claude Vision이 이미지를 분류·분석하고, 한국어 썸네일(JPG)과 쿠팡 상세페이지(HTML)를 동시에 생성하는 기능.

직접 스크래핑 대신 이미지 업로드 방식을 채택한 이유: 1688은 Alibaba 계열 anti-bot으로 직접 크롤링을 차단하며, 이미지 업로드 → AI 분석 방식이 오히려 출력 품질이 높고 인프라가 이미 갖춰져 있음.

---

## 전체 파이프라인

```
[셀러] 1688 이미지 N장 업로드
        ↓
[Step 1] Supabase Storage 업로드 (원본 보관)
         경로: 1688-import/{sessionId}/original/{filename}
        ↓
[Step 2] Claude Vision — 이미지 분류 API
         각 이미지를 4가지로 태깅:
         main_product / lifestyle / infographic / size_chart
        ↓
[Step 3] 사용자가 썸네일 후보(main_product) 확인 + 필요 시 변경
        ↓
[Step 4] Claude Vision — 한국어 콘텐츠 생성 API
         분류된 이미지 전체를 보고 생성:
         · 상품명 (쿠팡 SEO 최적화)
         · 핵심 셀링포인트 3개 (icon, title, description)
         · 상세 설명
         · 스펙 테이블
        ↓
[Step 5a] 썸네일 생성 (Sharp — AI 없음)
          main_product 이미지 500×500 크롭
          하단 20% 반투명 흰 배경 + SVG 한국어 상품명 오버레이
          coupang-constraints.ts 규격 보정 (min 500px, max 5000px, max 10MB)

[Step 5b] 상세페이지 생성 (html-builder.ts 재사용)
          Step 4 콘텐츠 + 원본 이미지들 → HTML 렌더링
        ↓
[결과] 썸네일 JPG 다운로드 + 상세페이지 HTML 복사
```

## UI 구조

진입점: `/listing` 페이지 → "1688에서 가져오기" 버튼 → `/listing/import-1688`

### 화면 1 — 업로드
- 드래그앤드롭 + 클릭 업로드 (JPG/PNG, 최대 20장)
- "분석 시작" 버튼 → Step 1+2 실행

### 화면 2 — 분류 확인 + 썸네일 선택
- 이미지 그리드, 각 이미지에 태그 배지 표시
- `main_product` 이미지에 썸네일 후보(★) 표시
- "변경" 버튼으로 다른 이미지를 썸네일로 교체 가능
- "한국어 콘텐츠 생성" 버튼 → Step 4+5 실행

### 화면 3 — 결과 미리보기
- 좌: 썸네일 500×500 미리보기
- 우: 상세페이지 HTML 미리보기
- 하단: [썸네일 JPG 다운로드] [상세페이지 HTML 복사]

기존 `CoupangAutoRegisterPanel` UI 패턴을 따름.

## API 설계

### `POST /api/listing/import-1688/classify`

```ts
// 입력
{ imageUrls: string[] }  // Supabase 업로드 완료된 URL 목록

// 출력
{
  images: Array<{
    url: string
    type: 'main_product' | 'lifestyle' | 'infographic' | 'size_chart'
  }>
}
```

**프롬프트 (Claude Vision):**
```
이미지들을 분류해주세요.
각 이미지에 대해 다음 중 하나로 태깅:
- main_product: 단독 상품컷 (흰/단색 배경)
- lifestyle: 사용 장면
- infographic: 텍스트/수치가 포함된 설명 이미지
- size_chart: 사이즈/규격표

JSON 배열로만 반환. 예: [{"index":0,"type":"main_product"}, ...]
```

### `POST /api/listing/import-1688/generate`

```ts
// 입력
{
  images: Array<{ url: string; type: string }>
  thumbnailUrl: string  // 사용자가 확정한 썸네일 이미지 URL
}

// 출력
{
  thumbnailUrl: string     // Supabase Storage URL (1688-import/{sessionId}/thumbnail.jpg)
  detailPageHtml: string   // html-builder 출력 HTML
}
```

**프롬프트 (Claude Vision, 기존 detail-page 프롬프트 variant 추가):**
```
업로드된 1688 상품 이미지들을 보고 한국 쿠팡 셀러용 상품 정보를 생성하세요.
중국어가 보이면 한국어로 의미를 파악해 작성하세요.

반환 형식 (DetailPageContent 타입):
{ headline, subheadline, sellingPoints[{icon,title,description}], specs[{label,value}] }
```

## 파일 구조

### 신규 파일
```
src/app/listing/import-1688/
  page.tsx                          ← 3단계 화면 흐름 (업로드→분류→결과)

src/app/api/listing/import-1688/
  classify/route.ts                 ← Step 2 AI 분류
  generate/route.ts                 ← Step 4+5 콘텐츠 생성 + 썸네일 렌더링

src/components/listing/import1688/
  ImageDropzone.tsx                 ← 업로드 드래그앤드롭
  ClassificationGrid.tsx            ← 이미지 태그 그리드 + 썸네일 변경
  ResultPreview.tsx                 ← 썸네일 + 상세페이지 나란히 미리보기

src/lib/listing/
  import-1688-thumbnail.ts          ← Sharp SVG 오버레이 로직
```

### 수정 파일
```
src/app/listing/page.tsx            ← "1688에서 가져오기" 버튼 추가
src/lib/ai/prompts/detail-page.ts   ← 1688용 프롬프트 variant 추가
```

### 재사용 (변경 없음)
```
src/lib/detail-page/html-builder.ts
src/lib/image/coupang-constraints.ts
```

### Supabase Storage
```
1688-import/{sessionId}/original/{filename}   ← 업로드 원본
1688-import/{sessionId}/thumbnail.jpg         ← 생성된 썸네일
```

`sessionId`: 업로드 시작 시 클라이언트에서 `crypto.randomUUID()`로 생성. 세션 간 파일 충돌 방지 및 임시 파일 정리 기준으로 사용.

## 에러 처리

| 상황 | 처리 |
|------|------|
| 분류 결과에 main_product 없음 | 첫 번째 이미지를 썸네일 후보로 fallback, 사용자에게 안내 |
| AI 콘텐츠 생성 실패 | 재시도 버튼 노출, 부분 결과 있으면 표시 |
| 이미지 규격 미달 (< 500px) | coupang-constraints가 자동 보정 |
| 20장 초과 업로드 | 업로드 단계에서 클라이언트 측 차단 |
