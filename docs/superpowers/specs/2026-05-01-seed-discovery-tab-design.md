# 시드 발굴 탭 설계 (Seed Discovery Tab)

## 개요

소싱 대시보드에 **🌱 시드 발굴** 탭을 신규 추가한다.  
채널 방법론(돈버는하마 유튜브)을 기반으로 카테고리 선택 → 키워드 자동 분석 → 검증 → 30개 확정 → 도매꾹 탭으로 이동하는 7단계 워크플로우를 제공한다.

## 배경 및 목적

- 현재 도매꾹 탭은 수동 수집 + 분석 중심이며, 시드 키워드 발굴을 위한 체계적인 워크플로우가 없음
- 채널 핵심 원칙: "키워드 먼저, 상품 나중" — 카테고리에서 소비자 실제 검색어를 역추출해야 함
- 시드 발굴 워크플로우는 다단계·멀티세션 구조이므로 도매꾹 탭과 분리된 전용 공간이 필요

## 채널 방법론 근거

| 기준 | 채널 영상 |
|---|---|
| 카테고리 → 키워드 역추출 | `[도매꾹 위탁판매 전에 알아야 할 팔리는 상품 소싱 방법 (2025-07-09)]` |
| 월 검색량 3,000~30,000 | `[잘 팔리는 상품 소싱? 쿠팡, 스마트스토어 키워드 찾는법 (2026-03-10)]` |
| 경쟁상품수 500개 미만 | `[쿠팡, 스마트스토어 경쟁이 적은 상품 소싱하는 방법 (2025-06-11)]` |
| 상위 리뷰 50개 미만 | `[쿠팡강연유출 — 로켓그로스 억대셀러 소싱법 (2025-09-29)]` |
| 위탁 마진 30% 이상 | `[자본금이 없는데 그냥 위탁만 해도 돈 벌 수 있을까요? (2025-09-01)]` |
| KC·시즌·50cm 회피 | `[로켓그로스 이런거 팔면 망합니다 (2025-11-24)]`, `[그로스 보관료 (2025-03-27)]` |
| 위너 확정 기준 5개 | `[쿠팡강연유출 — 로켓그로스 억대셀러 소싱법 (2025-09-29)]` |

## UI 구조

### 탭 재구성 (Before → After)

```
Before: 니치소싱 | 도매꾹 | 코스트코 | 발굴 메모 | 마진계산기 | 키워드 목록
After:  니치소싱 | 🌱 시드 발굴 | 도매꾹 | 코스트코 | 발굴 메모 | 마진계산기 | 키워드 목록
```

## 7단계 파이프라인

### Step 1 — 카테고리 선택 [사용자]

체크박스로 카테고리 선택 → 시드 키워드 자동 매핑

| 카테고리 | 자동 매핑 시드 키워드 (예시) |
|---|---|
| 생활용품 | 수납함, 정리함, 욕실용품, 방향제, 발매트 |
| 문구/사무 | 파일홀더, 메모지, 필통, 볼펜, 인덱스탭 |
| 반려동물 | 배변패드, 강아지간식, 고양이모래, 급수기 |
| 차량용품 | 차량방향제, 핸들커버, 차량거치대, 세차용품 |
| 가구/인테리어 | 선반, 수납장, 쿠션, 데코소품, 벽시계 |

---

### Gate 0 — 회피 하드 차단 [자동]

채널 방법론상 가장 먼저 실행하는 게이트. **상표권은 이 단계에서 처리하지 않음** (Step 5 KIPRIS로 이관).

| 차단 항목 | 판단 기준 |
|---|---|
| KC인증 필수 | 전기·전자, 유아용품, 식품, 유아의류 카테고리 |
| 시즌 한정 | 크리스마스, 설, 추석, 할로윈 키워드 포함 |
| 부피 50cm 초과 | 도매꾹 상품 스펙 최장변 기준 |

---

### Step 2 — 검색량·경쟁상품수 [자동]

기존 인프라 재사용 (`lib/niche/naver-shopping.ts`, `lib/naver-ad.ts`).

```
시드 키워드
  → naverShop.getSuggestions(hint)  // 네이버 자동완성 (인증 불필요)
  → expandKeywords(suggestions)     // 네이버 Ad API 검색량 조회
  → 필터: 3,000 ≤ searchVolume ≤ 30,000
  → naverShop.search(keyword)       // 경쟁상품수 조회
  → 필터: competitorCount < 500
```

---

### Step 3 — 쿠팡 상위 리뷰수 입력 [사용자 — 스킵 불가]

- "쿠팡↗" 버튼으로 해당 키워드 쿠팡 검색 새 탭 오픈
- 상위 3개 상품 중 리뷰수가 가장 많은 값을 숫자로 입력
- 입력값 ≥ 50 이면 자동 탈락 (해당 키워드는 진입 장벽이 높음)
- **미입력 행이 있으면 다음 단계 버튼 잠금**

채널 근거: 이 단계를 건너뛰면 경쟁 불가 상품을 위탁 등록하는 실수 발생 (`[초보가 소싱이 어려운 이유 (2025-11-17)]`)

---

### Step 4 — 도매꾹 매칭 + 마진 계산 [자동]

- 리뷰 통과 키워드로 `DomeggookClient.getAllItems({ keyword })` 호출
- 위탁 마진: `(시장가 - 도매가) / 시장가`
- 1688 예상 마진: 내부 설계 기준 (채널 직접 언급 없음, 40~60% 달성 가능 범위에서 유추)
  - 1688 마진 40% 미달 시 **탈락이 아닌 ⚠️ 경고 표시**
- 위탁·1688 마진 동시 표시 (사입 전환 가능성 판단용)

---

### Gate 1 — 마진 탈락 [자동]

각각 독립 탈락 조건:
- 위탁 마진 **30% 미달** → 즉시 탈락 (채널 직접 근거)
- 1688 예상 마진 40% 미달 → ⚠️ 경고 (탈락 아님, 내부 기준)

---

### Step 5 — KIPRIS 상표권 검사 [자동 + 부분 수동]

- 키워드 문자 상표 자동 검색 (기존 `kipris-client.ts` 재사용)
- 충돌 없음 → 자동 통과
- 유사 상표 발견 → ⚠️ 경고 + "확인 완료" 체크박스 표시 (사용자 수동 확인)
- 도형·캐릭터 유사성은 자동 판단 불가 → KIPRIS 링크 제공

---

### Step 6 — 시드 점수 산출 [자동]

**내부 설계 기준 (채널 방법론에 배점 없음)**

```
시드점수 = 경쟁(30) + 검색량(25) + 리뷰(25) + 마진(20) = 100점
```

| 항목 | 배점 | 방향 | 기준 |
|---|---|---|---|
| 경쟁상품수 | 30점 | **적을수록 고점** | 100개 미만 = 30점, 500개 = 0점 선형 |
| 월검색량 | 25점 | **역U형** (중간값 최고점) | ~15,000 = 25점, 3,000/30,000 = 12점 |
| 쿠팡 상위 리뷰수 | 25점 | **적을수록 고점** | 0개 = 25점, 50개 = 0점 선형 |
| 마진율 | 20점 | **높을수록 고점** | 30% = 0점, 60% = 20점 선형 |

등급: S(85+), A(70+), B(55+), C(40+), D(미만)

채널이 경쟁상품수를 1순위로 강조하므로 배점 최고 (`[초보가 소싱이 어려운 이유 (2025-11-17)]`).

---

### Step 7 — 30개 확정 [사용자]

- 점수 상위 정렬 후 체크박스로 30개 선택
- "도매꾹에 추가" 버튼 → 확정

**완료 후 다음 단계:**
- 키워드 30개 → SKU 30~50개 위탁 등록
- 2주 운영
- 위너 확정 기준 (채널 억대셀러 강연 5개):
  - 클릭 100+, 전환율 1.5%+, ROAS 250%+, 판매 5건+, 별점 4.0+
- 위너 5~10개 확정 → 1688 사입 발주
- 이 플로우를 2~3회 반복하여 총 80~100 SKU 달성

## 탭 간 연동 구조

### 단방향 흐름 (시드 발굴 → 도매꾹)

```
Step 7 확정
  → POST /api/sourcing/seed-discover/confirm
  → sourcing_items UPSERT (seed_keyword, seed_score, seed_session_id)
  → seed_sessions 상태 = 'confirmed'
  → 도매꾹 탭 리프레시 + 토스트 알림
```

### 역방향 참조 (도매꾹 → 시드 발굴)

- 2주 후 위너 확정 → `seed_sessions.winner_count` 업데이트
- 시드 발굴 세션 히스토리에서 "이 세션 위너 N개" 확인 가능
- 다음 시드 발굴 시 패턴 참고 용도

### 엣지 케이스

| 상황 | 처리 |
|---|---|
| 도매꾹에 이미 있는 상품 재추가 | `ON CONFLICT (item_no) DO UPDATE` — 기존 데이터 유지, seed 정보만 추가 |
| 세션 여러 번 실행 | `seed_session_id`로 구분, 세션별 독립 조회 가능 |
| 위너 역참조 | `winner_count` 통해 시드 세션 효과 측정 |

## 도매꾹 탭 변경 사항

### 신규 필터 (필터 툴바)

| 필터 | 동작 |
|---|---|
| 🌱 시드만 보기 | `seed_score IS NOT NULL` |
| 시드 제외 | `seed_score IS NULL` |
| 시드점수 N 이상 | `seed_score >= N` (숫자 입력 필드) |

### 신규 정렬

- 퀵 정렬 버튼: `🌱 시드점수↓` 추가
- 컬럼 헤더 클릭 정렬: `seed_score` 컬럼 추가

### 시드 상품 구분 표시

- 행 왼쪽 보라색 구분선 (`border-left: 3px solid #a78bfa`)
- 상품명 아래 `🌱 시드 YYYY-MM-DD` 배지
- `seed_keyword` 컬럼 (보라색)
- `seed_score` 컬럼 (보라색, 등급 표시)

## DB 스키마 변경

### sourcing_items 컬럼 추가 (3개)

```sql
ALTER TABLE sourcing_items ADD COLUMN
  seed_keyword    TEXT     DEFAULT NULL, -- "수납함"
  seed_score      SMALLINT DEFAULT NULL, -- 0~100
  seed_session_id UUID     DEFAULT NULL; -- seed_sessions FK
```

### seed_sessions 신규 테이블

```sql
CREATE TABLE seed_sessions (
  id           UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id      UUID        NOT NULL,
  categories   TEXT[],                        -- ['생활용품', '문구/사무']
  status       TEXT        DEFAULT 'in_progress', -- in_progress|confirmed
  step         SMALLINT    DEFAULT 1,          -- 현재 진행 단계 1~7
  state_json   JSONB,                          -- 임시저장 데이터
  confirmed_at TIMESTAMPTZ,
  winner_count SMALLINT    DEFAULT 0,
  created_at   TIMESTAMPTZ DEFAULT now()
);
```

## 내부 기준 명기

아래 기준은 채널 영상 직접 근거 없이 설계팀이 유추한 내부 기준이다:

- **1688 예상 마진 40%**: 채널이 "40~60% 달성 가능"이라고 언급한 범위에서 유추. 탈락 기준이 아닌 경고 기준으로 사용.
- **시드 점수 배점 비율**: 채널이 임계값(500개, 3,000~30,000 등)만 제시했으며 상대 가중치는 설계팀 결정. 경쟁상품수를 채널이 1순위로 강조하므로 배점 최고(30점).
- **카테고리별 시드 키워드 목록**: 채널 제시 우선 카테고리 5개 기반으로 시드를 사전 정의. 채널이 구체적 키워드 목록을 제시하지는 않음.
