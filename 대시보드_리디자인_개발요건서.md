# SmartSellerStudio 통합 대시보드 리디자인 개발 요건서

**문서 버전:** 1.0  
**작성일:** 2026-04-12  
**작성자:** AI 개발 어시스턴트  
**프로젝트:** SmartSellerStudio  
**대상 모듈:** `/dashboard` (메인 대시보드)

---

## 1. 개요

### 1.1 배경

기존 대시보드는 도매꾹 단일 채널 중심으로 설계되어 있었으나, 소싱 채널이 **도매꾹 + 코스트코 + 니치소싱**으로 확장됨에 따라 3채널 통합 관점의 대시보드로 전면 리디자인이 필요해졌다.

### 1.2 목적

3개 소싱 채널의 핵심 지표를 한눈에 파악할 수 있는 통합 대시보드를 제공하여, 채널별 성과 비교와 빠른 의사결정을 지원한다.

### 1.3 변경 범위

| 구분 | 파일 경로 | 변경 유형 |
|------|-----------|-----------|
| API 엔드포인트 | `src/app/api/dashboard/summary/route.ts` | **신규 생성** |
| 대시보드 UI | `src/components/dashboard/DashboardClient.tsx` | **전면 재작성** |
| 페이지 라우트 | `src/app/dashboard/page.tsx` | 변경 없음 (기존 유지) |

## 2. 기능 요건

### 2.1 통합 스코어보드 (상단 고정 영역)

페이지 최상단에 항상 노출되는 5칸 그리드 형태의 핵심 KPI 카드.

| # | 지표명 | 데이터 소스 | 표시 형식 | 비고 |
|---|--------|------------|-----------|------|
| 1 | 전체 소싱 상품 | 도매꾹 + 코스트코 합산 | `N개` | 채널별 내역 서브텍스트 |
| 2 | 주문 (7일) | 쿠팡 Orders API | `N건` | 신규 주문 건수 서브텍스트 |
| 3 | 매출 (7일) | 쿠팡 Orders API | `N만원` | 취소/반품 제외 |
| 4 | 평균 마진율 | 도매꾹·코스트코 가중평균 | `N%` | 채널별 마진 서브텍스트 |
| 5 | 법적 이슈 | 도매꾹 legal_status | `N건` | 주의 건수 서브텍스트 |

### 2.2 채널 탭 시스템 (탭 기반 멀티뷰)

스코어보드 하단에 4개 탭을 배치하여, 탭 전환 시 해당 영역만 교체된다.

#### 탭 1: 전체 요약

2×2 그리드 레이아웃.

| 영역 | 내용 |
|------|------|
| 좌상 — 채널별 마진 비교 | 도매꾹·코스트코 마진율을 수평 바 차트로 비교. 목표 마진선(도매꾹 10%, 코스트코 13%) 시각 표시 |
| 우상 — 주문/매출 현황 | 7일 기준 총 주문·총 매출·신규 주문 요약 |
| 좌하 — 주의 필요 | KC 인증 누락, 상표/과장광고 주의, 미처리 신규주문 등 액션 아이템 |
| 우하 — 니치 트렌드 | 추적 키워드 수, 평균 점수, Top 키워드 태그 (등급별 색상) |

#### 탭 2: 도매꾹

| 구성 요소 | 설명 |
|-----------|------|
| KPI 4종 | 추적 상품 수, 평균 마진율, 법적 차단 건수, 법적 주의 건수 |
| 7일 판매 Top 5 | 상품명, 도매가, 판매 건수, 마진율 배지 (20%↑ 녹색, 10%↑ 주황, 미만 빨강) |
| 마지막 수집 시각 | 하단 우측 타임스탬프 |

#### 탭 3: 코스트코

| 구성 요소 | 설명 |
|-----------|------|
| KPI 4종 | 수집 상품 수, 평균 스코어, 평균 마진율, S/A 등급 수 |
| 등급 분포 바 | S(80+)/A(65+)/B(50+)/C·D(50 미만) 비율을 컬러 바로 시각화 |
| 등급 범례 | 각 등급의 개수 및 기준 점수 표시 |
| 정책 안내 | 목표 순이익률 13%, 채널 수수료(네이버 6%, 쿠팡 11%) |

#### 탭 4: 니치소싱

| 구성 요소 | 설명 |
|-----------|------|
| KPI 3종 | 추적 키워드 수, 평균 점수, Top 키워드 수 |
| 키워드 Top 5 | 키워드명, 점수, 등급 배지 (S=녹색, A=파랑, 그 외 회색) |
| CTA 버튼 | "니치 키워드 분석하러 가기" → `/sourcing` 이동 |

### 2.3 퀵 액션 (하단)

4칸 그리드로 주요 작업 바로가기 제공.

| 액션 | 설명 | 링크 | 색상 |
|------|------|------|------|
| 도매꾹 소싱 | 드롭쉬핑 상품 수집 | `/sourcing` | #2563eb |
| 코스트코 소싱 | 사입 상품 분석 | `/sourcing` | #be0014 |
| 니치 분석 | 시장 트렌드 탐색 | `/sourcing` | #7c3aed |
| 매출 분석 | 주문/매출 현황 | `/orders` | #16a34a |

## 3. API 설계

### 3.1 GET /api/dashboard/summary

3채널 통합 요약 데이터를 단일 요청으로 반환한다.

**요청:** 파라미터 없음  
**캐싱:** `force-dynamic` (매 요청마다 최신 데이터)

**응답 구조:**

```json
{
  "success": true,
  "data": {
    "overview": {
      "totalProducts": 1250,
      "avgMarginDomeggook": 12.3,
      "avgMarginCostco": 15.1,
      "legalBlocked": 5,
      "legalWarning": 12
    },
    "channels": {
      "domeggook": {
        "products": 800,
        "avgMargin": 12.3,
        "topSellers": [
          {
            "title": "상품명",
            "sales7d": 45,
            "marginRate": 18.5,
            "scoreTotal": 72.0,
            "priceDome": 15000
          }
        ],
        "legalBlocked": 5,
        "legalWarning": 12,
        "lastCollected": "2026-04-12T09:00:00Z"
      },
      "costco": {
        "products": 450,
        "avgScore": 62.5,
        "avgMargin": 15.1,
        "gradeDistribution": { "S": 12, "A": 45, "B": 120, "CD": 273 },
        "lastCollected": "2026-04-12T08:30:00Z"

          }
        ]
      }
    }
  }
}
```

### 3.2 DB 쿼리 전략

모든 쿼리는 `Promise.all`로 병렬 실행하여 응답 시간을 최소화한다.

| # | 대상 테이블 | 쿼리 목적 |
|---|------------|-----------|
| 1 | `sales_analysis_view` | 도매꾹 총 상품수 + 법적이슈 집계 |
| 2 | `sales_analysis_view` | 도매꾹 Top 5 판매 (sales_7d DESC) |
| 3 | `sales_analysis_view` | 도매꾹 평균 마진율 |
| 4 | `costco_products` | 코스트코 총 상품수 + 평균 스코어 |
| 5 | `costco_products` | 코스트코 등급 분포 (FILTER 집계) |
| 6 | `costco_products` | 코스트코 평균 마진율 (시장가 대비 역산) |
| 7 | `niche_keywords` | 니치 추적 키워드 수 + 평균 점수 |
| 8 | `niche_keywords` | 니치 Top 5 키워드 |

### 3.3 기존 API 활용

| API | 용도 | 비고 |
|-----|------|------|
| `GET /api/orders/coupang` | 7일 주문/매출 데이터 | 기존 엔드포인트 그대로 활용 |

## 4. UI/UX 설계

### 4.1 디자인 원칙

- 기존 SmartSellerStudio 디자인 시스템 유지 (accent: `#be0014`, 인라인 style)

```

│  헤더 (로고 + 네비게이션)                  │

│ 전체 │ 주문 │ 매출 │ 마진 │  법적이슈       │  ← 통합 스코어보드 (고정)
│ 상품 │(7일)│(7일)│ 율  │                  │
├────┴────┴────┴────┴────────────────────┤
│ [전체요약] [도매꾹] [코스트코] [니치소싱]   │  ← 채널 탭
├─────────────────────────────────────────┤
│                                         │
│  탭별 콘텐츠 영역                         │  ← 탭 전환 시 교체

│도매꾹│코스트코 │  니치   │  매출분석       │  ← 퀵 액션
│소싱  │ 소싱   │  분석   │               │

```

## 5. 데이터 흐름

```
[페이지 로드]
    │
    ├─→ GET /api/dashboard/summary  ──→  DB 병렬 쿼리 (8개)
    │       └─→ setData(response)
    │
    └─→ GET /api/orders/coupang     ──→  쿠팡 API
            └─→ setOrderStats(response)
    │
[렌더링]
    │
    ├─→ 통합 스코어보드: data.overview + orderStats
    ├─→ 전체요약 탭: data.channels.* + orderStats
    ├─→ 도매꾹 탭: data.channels.domeggook
    ├─→ 코스트코 탭: data.channels.costco
    └─→ 니치 탭: data.channels.niche
```

## 6. 비기능 요건

| 항목 | 요건 |
|------|------|
| 에러 처리 | API 실패 시에도 대시보드 UI는 정상 렌더링 (빈 상태 표시) |

| 성능 | DB 쿼리 8개 `Promise.all` 병렬 실행 |
| 타입 안전성 | TypeScript strict 모드 통과, ESLint 에러 0건 |
| 디자인 일관성 | 기존 accent 색상(`#be0014`), 인라인 style 패턴 유지 |

## 7. 기존 대시보드 대비 변경 사항

| 항목 | Before (기존) | After (변경 후) |
|------|--------------|----------------|
| API | `/api/sourcing/analyze` + `/api/orders/coupang` 개별 호출 | `/api/dashboard/summary` 통합 API + `/api/orders/coupang` |
| 요약 카드 | 4칸 (추적상품, 주문, 매출, 법적이슈) | 5칸 (전체상품, 주문, 매출, **평균마진율**, 법적이슈) |
| 콘텐츠 | Top 5 판매 + 주의 필요 (고정) | **탭 기반 4개 뷰** (전체/도매꾹/코스트코/니치) |
| 채널 커버리지 | 도매꾹 단일 | **도매꾹 + 코스트코 + 니치** 3채널 |
| 마진 비교 | 없음 | **채널별 마진 바 차트** (목표선 포함) |
| 코스트코 정보 | 없음 | **등급 분포 시각화** (S/A/B/C·D) |
| 니치 정보 | 없음 | **Top 키워드 + 등급 태그** |
| 퀵 액션 | 소싱수집, 상세페이지, 상품등록, 매출분석 | **도매꾹소싱, 코스트코소싱, 니치분석**, 매출분석 |
